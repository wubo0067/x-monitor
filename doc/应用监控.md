# 应用监控

## 应用定义

应用可以认为是一个中间件服务、一个数据库服务。应用本身可以是一个进程，也可以是一组进程。x-monitor收集应用对应的进程指标求和后输出。应用名是唯一的标识，由管理员定义

## 配置过滤规则

应用采集配置在collector_plugin_appstatus = {}节点中。

1. 基础配置
   1. enable：是否激活应用采集。
   2. update_every：应用指标采集的时间间隔。
   3. update_every_for_app：每一条规则对应一个应用，应用启动后在该时间间隔内会匹配上。
   4. update_every_for_filter_rules：规则更新时间间隔，到期读取filter_sources的内容。

3. 匹配规则
4. 附加keys
5. 对应进程的方式
